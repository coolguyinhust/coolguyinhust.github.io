方法 说明
final void wait() 释放CPU的行使权，释放锁。线程执行wait()时，进入等待池（Waiting Pool）,同时失去了对象的机锁—暂时的，wait后返还对象锁。当前线程必须拥有当前对象锁，所以wait()必须在synchronized block中。
final void notify()/notifyAll() 唤醒等待池中的第一个/所有线程，必须拥有相同对象的锁，否则抛出IllegalMonitorStateException异常。
static void stop() 停止线程，不推荐使用
void start() 启动线程
void run() 如果该线程是使用Runnable对象构造的，则调用Runnable的run()；否则，该方法不执行任何操作并返回。
static void sleep() 在指定的毫秒数内让当前线程休眠（停止执行）。释放CPU的行使权，不释放锁。
static void yield() 暂停当前对象的执行，并执行其它线程
void join() 等待该线程中断
void setPriority(int newPriority ) 更改线程的优先级
int getPriority() 返回线程的优先级

线程的优先级：

代表线程的重要程度，有多个线程处于等待获取CPU时间时，调度系统根据线程的优先级来决定给谁分配CPU时间，优先级高的获得的机会更大，但优先级低的线程也有机会（小一些）获得。

利用Thread.interrupt()方法和机制

Java中断机制是一种协作机制，也就是说通过中断并不能直接终止另一个线程，而需要被中断的线程自己处理中断。这好比是家里的父母叮嘱在外的子女要注意身体，但子女是否注意身体，怎么注意身体则完全取决于自己。

Java中断模型也是这么简单，每个线程对象里都有一个boolean类型的标识（不一定就要是Thread类的字段，实际上也的确不是，这几个方法最终都是通过native方法来完成的），代表着是否有中断请求（该请求可以来自所有线程，包括被中断的线程本身）。例如，当线程t1想中断线程t2，只需要在线程t1中将线程t2对象的中断标识置为true，然后线程2可以选择在合适的时候处理该中断请求，甚至可以不理会该请求，就像这个线程没有被中断一样。

java.lang.Thread类提供了几个方法来操作这个中断状态，这些方法包括：

中断线程。
public static boolean interrupted() 测试当前线程是否已经中断。线程的中断状态由该方法清除。换句话说，如果连续两次调用该方法，则第二次调用将返回 false（在第一次调用已清除了其中断状态之后，且第二次调用检验完中断状态前，当前线程再次中断的情况除外）。
public boolean isInterrupted() 判断线程是否已经中断。线程的中断状态不受该方法的影响。
public void interrupt() 中断线程。

假设某个线程要不停地处理某件事情(比如 i 一直自增)，但是还有个要求：在处理事情前，先要检查下这个线程是否被中断，如果已经被中断，处理就应该结束。

线程的生命周期

线程的生命周期一共5种状态，new ,runnable , running , blocked ,dead.

1) 新建 new

MyThread thread = new MyThread(); 新生的线程已经有了自己的内存空间，但没有运行，还不是活的(not alive)

2)就绪 runnable

start()启动线程，正在等待CPU分配时间片，线程处于就绪队列，状态alive，但还没执行。

3）运行 running

线程获得CPU资源正在执行run()。

4）阻塞 blocked

由于某种原因导致正在运行的线程让出自己的CPU并暂停自己的执行，即进入阻塞状态。

  • 正在睡眠： 用sleep()使线程进入的睡眠方式，在指定的时间过去后就进入就绪状态。
  • 正在等待： 调用wait()。可调用notify()回到就绪状态。
  • 被另一个线程所阻塞： 调用suspend()，可用resume()恢复。

5）死亡 dead

  • run()方法终止后。
  • stop()方法让一个线程终止运行。

什么是守护线程

后台运行的线程。所有线程结束时，守护线程才结束，不需要手动操作。

线程组 ThreadGroup

每一个ThreadGroup包含一组子线程或一组子线程组。方便线程管理。它与线程池不同，线程池是为了管理线程的生命周期，复用线程，减少创建和销毁线程的开销。

Thread.currentThread().getThreadGroup();//获得当前线程的线程组。

Java运行我们对一个线程组的所有线程同时进行操作。

当前线程的副本：ThreadLocal

为每个使用该变量的线程提供独立的变量副本，每个线程可以改变自己的副本，不影响其它线程对应的副本。

线程的异常处理

run()不允许throw exception，所有异常必须在run方法内进行处理。
