algorand

发表于 2018-11-22

​ 现在区块链饱受诟病并一直不被大众接收，除了难以理解之外，其自身原因大抵是因为各种共识算法存在的弊端，POW耗财耗电，POS又有悖于完全去中心化的初衷，易被“大鳄”控制，小散的积极性不高；

algorand是一种新的方案，它类似于当前新股申购的中签方式（抽签），让人人都有机会发布区块链；当然它解决了很多其它问题，比如怎样防止adversary的攻击…

algorand的最终目的就是尽可能做到完全随机。

简单来说，就是每个节点在一轮中的每个step,都得到一个随机数（用的是VRF可验证函数），如果随机数满足要求，则成为verifier。（在step=1时，verifier和potential leader一致；）

BA*

下面介绍一下BA*共识：

假设：消息接收的时延有限；

​ 每一个step中 honest verifiers>=2/3 verifiers；

​ r是第几轮，s是第几步；Qr是r轮的种子；

​ vi表示H(leader,block),即能验证是某个leader发出的block

step1 :

对0和1之间的一个数，0.H(SIGi(r,s,Qr-1))≤p1发生的概率为p1，称所有满足此条件的用户为“potential leader”（也是这一步的“验证者”）。p使得以1-10^-18的概率，在所有“潜在领导者”中，至少有一个是诚实的。

potential leader生成自己的区块和signed message，签名完后销毁自己的密钥（即使敌手控制了该potential leader，也无法让他发送新的message）；并广播block和message；

step2：

verifiers从收到的msg(r,1)里面，找到最小的那个值，把那个当成leader，检查合法性；广播msg(r,2)；

step3：

收到的msg(r,2)中，如果有超过2/3的消息的vi相同，广播msg(r,3)

step4:

收到的msg(r,3)中，

​ 如果有超过2/3的消息的vi相同，令bi=0;(表示“我”认为区块有效)；

​ 超过1/3的vi相同且vi存在，令vi=v，bi=1;

​ 否则，令vi=null，bi=1;

广播msg(r,4)=（signed(bi),signed(vi)）;

step4之后进入循环验证阶段：

核心：bi为0还是1的数量谁先超过2/3；

0先超过2/3，表示生成区块有效；————ending condition 0

1先超过2/3，区块为空；————————-ending condition 1

在每个step s（s>=5且s<m）中，

  1. (s-2)mod 3==0，如果触发Ending Condition 0 或 1，则停止；否则等待ts后，收到的msg(r,s-1)中有超过2/3的bi是1，则将自己的bi置为1；否则置为0；
  2. (s-2)mod 3==1，如果触发Ending Condition 0 或 1，则停止；否则等待ts后，收到的msg(r,s-1)中有超过2/3的bi是0，则将自己的bi置为0；否则置为1；
  3. (s-2)mod 3==2，如果触发Ending Condition 0 或 1，则停止；否则等待ts后，收到的msg(r,s-1)中有超过2/3的bi是1，则将自己的bi置为1；收到的msg(r,s-1)中有超过2/3的bi是0，则将自己的bi置为0；否则，随机地置为0或1；

如果s<m前没有触发ending comdition，则区块也置为空。

进行多次循环的原因：

拜占庭共识要保证参与共识的诚实节点大于2/3，但随机选出的集合不能保证该条件。于是

  1. 进行多次的随机选取（循环），只要有一次参与共识的诚实节点大于 2/3，就能达成共识。
  2. 如果不进行多次随机选取（循环），则恶意节点每次把合法区块变成空区块的概率就会大大增加。
