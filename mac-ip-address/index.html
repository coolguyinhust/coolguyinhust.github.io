mac地址与IP地址的联系与区别

发表于 2018-10-05

参考知乎回答笔记

定义

MAC地址是烧录在网卡或者接口上的物理地址，具有二层意义和全球唯一性，一般不能被改变。

IP地址是网络中的主机或者三层接口在网络中的逻辑地址，在同一个网络内具有唯一性。

快递员要来找到张三给他送包裹，mac地址就是张三这个人，走到哪都是张三，世界上独一无二
ip地址就是他家地址某市某区某小区多少号，这地方任何人都可以住，可以是张三也可以不是张三
哪天张三搬家了，要找张三就得换一个地址，原来的地址住的也不是张三了。

不同主机之间是如何传输数据的？

同一网络内：

ARP协议：将目标主机的IP地址转换为对应主机的MAC地址，双方才能进行通信

通常主机网卡配置的是IP地址，二层以下数据帧的交换一般通过交换机或其他二层设备，需要用到主机间的MAC地址。

MAC地址表：交换机自己还要维护一张“MAC地址表”，用来记录目的MAC地址-端口的映射关系。

PC0要和PC1通信，只知道PC1的IP为192.168.0.2，于是PC0通过ARP Request广播方式，PC1是广播域的一员，会收到ARP，于是会以ARP Apply的方式告诉PC0自己的MAC地址，PC0会把PC1的ip-mac缓存在ARP table中

PC0就可以将PC1的IP地址转换为对应的MAC地址封装在数据帧中发给交换机，交换机自己通过维护一张“MAC地址表”，来判断你发送的这个数据帧的mac地址该从自己的哪个端口转发出去。

从上面的例子可以看出，IP地址实际并不起作用，在生成数据帧的时候就转化为MAC地址了，并不参与实际的通信过程。但是如果要通信的主机不在一个网络内，这时IP地址才真正发挥它的作用。

不同网络内：

路由器的两端分别为两个网络。数据想要从PC0到PC1，实际上经历了两个过程：

先从主机到路由器的F0/0，再从F1/0口到PC1.(这两个过程与同一网络中两个主机通信相同)

第一步，数据想从PC0到路由器，依然必须经历从IP地址到MAC地址的转换过程，而这个网关的MAC恰好就是F0/0口的MAC

现在查看路由器的路由表：

路由器收到数据包后会查看数据包的目的IP字段，我们是发送到192.168.1.1这台主机，看路由表发现正好有这个条目，到192.168.1.0/24要从F1/0这个端口发送出去。

Summary

IP地址只在不同网络间寻址才起作用，在同一个网络内，IP地址在发送端被转化为MAC地址进行寻址，而这种转化和交换的对应关系，依赖于ARP协议和MAC地址表。

数据链路层基于MAC地址转发数据帧，交换机设备维护”mac-port” table

网络层基于IP地址转发报文,路由设备维护”ip-port” table
