计算机网络复习（二）

发表于 2018-11-12

1968-1972:早期 ARPANET 验证了分组交换网络的有效性。

1972-1980: 越来越多的科研机构接入到ARPAnet,开始尝试解决多个不同网络的互连问题。

1973，提出解决网络互联问题的第一个思路，传输控制程序。

1973 – 1977，唯一的传输控制程序需要解决的问题很多：异构网络上的可靠传输问题、分组交换的路由问题、差错检测问题、……

1978-1980，形成TCP/IP协议族。

1980s， TCP/IP协议族被ARPNET正式采纳。

IP服务模型

采用数据报交换，每个分组自描述源/目的地址，沿途的路由器根据转发表转发分组；

IP分组设计

分组的分段与组装： （分段重组由目的主机完成）
问题：不同的网络存在不同的MTUs，最大传输单元(MTU, Maximum Transmission Unit )， 网络能够承载的最大的分组/数据帧。

层次化的IP地址划分 :

IP地址的表示

IP地址与掩码相与即获得其网络地址;

每个网段内，主机部分全为0的地址为网络地址(network address)，约定每个网段内主机部分全为1的地址为广播地址(broadcast address).

IP路由器

IP路由器的两大核心功能：

  • 转发: 将路由器输入端口收到的分组从正确的输出端口发送出去
  • 路由选择: 决定分组从源节点到达目的节点的路径路由选择算法

转发(数据平面)：获得一个分组, 查看其目的地址, 查询本地转发表, 将分组从输出端口发送出去

路由选择(控制平面)：转发表的构建通常由基于复杂分布式算法的路由协议完成

路由器周期性交互拓扑信息，通过路由算法构造路由表
路由表格式: <网络号, 下一跳>，路由表构造转发表

转发表格式: <网络号, 下一跳, 输出端口号>

（下面这个路由表其实包含了转发表的内容）

ARP：从IP地址到MAC地址的映射。

子网划分

传统A/B/C类网络地址划分基础上引入的新层次，同一个网络号的网段地址可以分为若干个子网。

子网掩码的1的位数是（网络号 + 子网号）的位数，0的位数是主机号的位数。

IP地址 与运算 子网掩码 = 子网地址（子网的网络号）

子网中不是所有的地址都可以作为主机地址：

  • 子网网络地址 (主机部分全0)
  • 子网广播地址 (主机部分全1)
  • 子网默认网关地址（缺省路由器地址）

CIDR

例：校园网地址 200.24.16.0/20

​ 电信系地址 200.24.16.0/23

就叫可变长的子网掩码，（网络号/子网掩码）

将两个或者多个子网组合起来，在路由表中形成单一路由记录的方法，又称为路由聚合或路由汇总。

最长前缀匹配准则(Longest Prefix Match)：当路由表中有多条条目可以匹配目的IP时，一般就采用掩码最长的一条作为匹配项并确定下一跳。

DHCP

自动为主机分配 IP的需求。

动态主机配置协议

ICMP

互联网控制消息协议

ping: 测试网络上某个主机是否可达的网络调试工具.

Ping 发 送一个 ICMP 回声请求 (type 8, code 0)消息给目的地，并报告是否收到所希望的 ICMP 回声应答

路由

路由选择算法： 生成路由表，为路由器转发IP分组找出适当的下一跳路由器。

路由选择协议：实现路由表中路由信息的动态更新，收到报文的操作…

自治系统：（AS）一个自治系统内部路由器之间能够使用动态的路由选择协议，及时地交换路由信息，精确地反映自治系统网络拓扑的当前状态。

自治系统内部的路由选择称为域内路由选择(intra-domain routing)；自治系统之间的路由选择称为域间路由选择(inter-domain routing)。

路由选择协议也分为两大类： 内部网关协议（IGP, Interior Gateway Protocol）、外部网关协议（EGP, Exterior Gateway Protocol）。

IGP

路由选择本质上是图论中寻找两个节点之间较低成本的路径的问题。

以路由器角色运行最短路径算法。

  1. 获取拓扑
  2. 计算路由
距离向量（RIP）

基于Bellman-Ford 算法：两两交换信息，根据接收到的信息更新到各节点的最短路径

每个节点与相邻节点交换当前自己到所有节点的距离；

初始路由信息：

A: (0, 1, 1, ∞, 1, 1, ∞) B: (1, 0, 1, ∞, ∞, ∞, ∞)
C: (1, 1, 0, 1, ∞, ∞, ∞) E: (1, ∞, ∞, ∞, 0, ∞, ∞)
F: (1, ∞, ∞, ∞, ∞, 0, 1)

B收到A的距离向量A: (0, 1, 1, ∞, 1, 1, ∞) 则更新自己的为

B: (1, 0, 1, ∞, 2, 2, ∞)

……

我们默认B收到A消息的同时，A会收到B的消息；且B几乎同时收到C D E F G的消息；（先后存在细微差别）

路由信息协议(RIP, Routing Information Protocol)

最大15跳；路由器通告到达网络的代价, 而不是到达其他路由器的代价

链路状态(OSPF)

每个节点洪泛（告诉所有节点）自己邻接点的消息。

OSPF协议，类似车载导航系统，将每条道路都收集到自己的数据库里。当计算A到C的最短路时，有以下选择：

A——–B————C

A——-D————E——–C

A———–B———–E———C

第一条路径最短，优先选择。如果遇到中途B严重堵车，可以提前切换到京沪高速，提高通行的效率。

OSPF比RIP强大的地方是，OSPF对整网的拓扑结构了如指掌，一旦某一条路径断了，可以及时选择备份链路，对通信的影响小。

那么OSPF又是怎样怎样得到A到C的航线的呢？

A会搜寻A自身的邻接点，搜到B，B又会把自身的邻接点信息广播出去，A就会收到。

相当于每个都手握一张残缺地图的部分，大家聚在一起，把各自的部分拼在一起就得到了所有路径！

官方说法：每个节点告诉所有其他节点与之直接相连的链路状态，通过Dijkstra算法计算路由。

开放最短路径优先路由协议：

链路状态报文LSP：

链路状态通告 (Link State Advertisement, LSA)
与该节点直接相邻的节点信息列表 <AdjacentNode, Cost>, 其中包括到这些邻节点的链路代价
一个链路状态报文可能包含多个LSAs

EGP

域内路由选择协议又称为内部网关协议 (Interior Gateway Protocol, IGP), 例如 OSPF, RIP

域间路由协议又称为外部网关协议 (Exterior Gateway Protocol, EGP), 例如 BGP。

核心思想: 向邻居网络通告整个路径而不是距离向量，各网络根据收到的路径向量更新自己的路由并且向其它网络转发

路径向量: 发送到每一个目的网络的路径向量。

边界网关协议（BGP）

BGP “发言人”路由器
在AS的边界负责向其它AS转发路径的路由器，每个AS至少有一个BGP“发言人”(Speaker)。

BGP会话

两个BGP发言人之间建立基于TCP的可靠传输

如果路径变化不大，则发送keep alive消息

如果路径发生了变化，则发送撤销withdrawn消息

更新路径信息时，采用增量更新的策略;

例如：

挑战：把底层网络低于要求的特性转变成为应用程序所需的高级服务

  • 不保证消息的可靠传送

字段：

  • 源端口和目的端口: 识别应用进程

  • 校验和: 选项, 计算整个报文和伪首部

可靠的字节流(TCP)

设计理念

TCP想要解决的问题：IP提供的是不可靠的传输，可能丢包，超时，乱序…

提供可靠传输的两个基本机制:
(1) ACK (确认) (2) 超时定时器

TCP采用改进的滑动窗口算法实现

  • 保证数据的可靠传送
  • 确保数据的有序传送
  • 流量控制(基于变化的AdvertisedWindow字段)

数据包格式（首部）

TCP不区分应用进程写入报文的边界, 采用数据段(Segment)为单位进行数据传输。

TCP首部：

标志位：（0或1）

SYN, FIN – 建立/终止TCP会话连接；

ACK – 有效回复时置为1

Seq: 发送的第一个字节序号

Ack: 期望收到的下一个字节的序号

AdWin: 接收方通知发送方的窗口大小

连接建立与终止

三次握手建立连接
​ client发送连接建立请求(SYN=1,Seq=x) 至server
​ Server回复确认Client前序请求(ACK=1,Ack=x+1)，同时请求(SYN=1,Seq=y)
​ Client回复确认Server前序请求(ACK=1,ACK=y+1)

四次握手断开连接Four-Way Wavehand
​ Client发送连接断开请求(FIN=1,Seq=u) 至server
​ Server回复确认client请求(ACK=1,Ack=u+1)，同时继续发送待传输数据(Seq=v)
​ Server等待上层应用数据传输完毕之后，Server重复确认前序请求(ACK=1,Ack=u+1)，同时发送连接断开请求

​ （FIN=1,Seq=w)
​ Client回复确认server请求(ACK=1,ACK=w+1,Seq=u+1)，继续等待2MSL时间后关闭

TCP滑动窗口算法

防止回绕：序号空间的大小是窗口空间大小的2倍；

然而, 分组的生命周期可能很长，于是引入MSL( Maximum Segment Lifetime)；

保持管道满载：发送方的窗口大小大于时延带宽积

拥塞控制

如何检测网络的拥塞? 以超时作为网络拥塞的标志。
如何调整发送速率? TCP维护一个Congestion Window(cwnd)变量。

TCP的发送方维护一个拥塞窗口cwnd，实际发送窗口不得超过cwnd。

网络拥塞时(分段传输超时)，cwnd减小；网络未拥塞时(分段成功传送)，cwnd增大。

接收窗口vs. 拥塞窗口：

接收窗口rwnd的描述单位是报文段中的字节序号(seq)；

拥塞窗口cwnd的描述单位是报文段的个数(segment)；

慢启动

初始窗口值偏大，容易形成突发流量导致拥塞，需要设计一种机制控制窗口启动阶段的cwnd。

  • TCP初始启动时，设置拥塞窗口cwnd为1
  • 慢启动阶段：以指数方式增加cwnd直到特定门限值ssthresh（ssthresh:slow start threshold）
  • 拥塞避免阶段：超过ssthresh后，cwnd加性增加；再次拥塞时，ssthresh值减半，重新进入慢启动阶段。

有时不是超时而是丢包，这种情况可以通过三次ACK确认，此时网络没有严重拥塞，检测到3次重复ACK后，不必等待超时计时器（快速重传），也没有必要进行慢启动（快速恢复），可以直接立即发起重传。
